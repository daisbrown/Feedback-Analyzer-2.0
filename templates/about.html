How It Works

This application provides a web interface to view and analyze user feedback that has been logged to JSONL files stored in an Azure Storage container. Access requires authentication via a local username and password.

Here's a summary of the process:

  • Authentication: Access to the application is protected by a local username/password system. User login status is managed using Flask sessions.
  • Containerization: The entire application runs as a Docker container based on an Ubuntu image, ensuring a consistent environment for deployment.
  • Web Framework: It utilizes Flask, a lightweight Python web framework, to handle web requests and render pages.
  • Azure Connection: It securely connects to a specific Azure Storage Account container using a Container SAS URL. This URL, which includes permissions and an expiry date, is provided via an environment file (azure.env) for security. The application requires SAS permissions for both List (to find files) and Read (to download file content).
  • Scheduled Data Fetching: A background scheduler (APScheduler) runs every 15 minutes within the application. This task scans the configured Azure container, including any subfolders, looking for files that start with feedback_log_ and end with .jsonl.
  • Data Processing & Storage: For each matching file found, the application downloads its content. It reads the file line by line, parsing each line as a JSON object. It extracts key fields like ID, timestamp, user details, feedback type, and content. This extracted data, along with the source Azure file path, is then inserted into a local SQLite database file (feedback.db).
  • Incremental Updates: To ensure efficiency and avoid duplicates, the application uses an INSERT OR IGNORE database command based on the feedback entry's unique id (which is set as the Primary Key). This means that even though all files are scanned, only feedback entries with IDs not already present in the database are actually added during each 15-minute cycle.
  • Web Interface: After logging in, the main page queries the SQLite database and displays the collected feedback data in a sortable, searchable table. It uses the Jinja2 templating engine (with Flask) and custom CSS for styling, including the company logo and a modern theme.
  • Features: Authenticated users can sort the data by clicking on most column headers and can filter the results by typing into the search bar, which checks multiple fields for matches.

The primary technologies involved are Python 3.9+, Flask, Flask-Session, Werkzeug (for hashing), SQLite3, APScheduler, the azure-storage-blob SDK, HTML5, CSS3, Jinja2, Gunicorn (as the web server within Docker), and Docker itself.
